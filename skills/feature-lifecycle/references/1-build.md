# Phase 1: Build (The Maker)

**Goal**: Implement the feature with high quality, aesthetics, and test coverage.

## Protocol

### 1. Pre-Flight (Understand & Plan)
**CRITICAL**: Do not write a single line of code until you have:
1.  **Domain Logic Check**: List the "Obvious" rules. (e.g., "Discounts can't be negative", "Start date must be before End date").
2.  **Architecture**: Define which files will be created/modified.
3.  **User Experience**: Define the "Happy Path" step-by-step.

### 2. Execute (The Iron Law)
*   **RED**: Write a failing test for the *smallest* logical unit.
*   **GREEN**: Write *only* enough code to pass.
*   **REFACTOR**: Clean up immediately.

**Stop & Think**: If you find yourself writing a lot of code without a test, **STOP**. Revert and write the test.
