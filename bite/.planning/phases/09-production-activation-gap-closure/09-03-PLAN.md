  <action>
Run the following gcloud command to enable automated backups and binary logging on the Cloud SQL instance:

```bash
gcloud sql instances patch bite \
  --project=ascent-web-260224-119 \
  --backup-start-time=02:00 \
  --retained-backups-count=7 \
  --enable-bin-log \
  --retained-transaction-log-days=7
```

This sets:
- Daily automated backups at 02:00 UTC (06:00 Oman local time)
- 7-day backup retention (per D-08 from Phase 8)
- Binary logging enabled (required for point-in-time recovery)
- 7-day transaction log retention for PITR window

If the command still fails with "Free Trial Instance" error, wait and retry later. The restriction will lift automatically.

After the patch command succeeds, verify the configuration:

```bash
gcloud sql instances describe bite \
  --project=ascent-web-260224-119 \
  --format="yaml(settings.backupConfiguration)"
```

Expected output:
```yaml
settings:
  backupConfiguration:
    enabled: true
    binaryLogEnabled: true
    startTime: "02:00"
    backupRetentionSettings:
      retainedBackups: 7
    transactionLogRetentionDays: 7
```

Note: The patch may cause a brief (seconds) instance restart as binary logging requires an InnoDB configuration change. This is normal and the service is not live with customers yet.
