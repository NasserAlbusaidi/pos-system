  <action>
    This task runs three gcloud operations sequentially. All values are concrete — no placeholders except the Sentry DSN and bucket name provided by Task 1.

    **Step 1 — Create GCS bucket if needed (per D-01):**

    If user said "create bite-pos-storage":
    ```bash
    gcloud storage buckets create gs://bite-pos-storage \
      --project=ascent-web-260224-119 \
      --location=us-central1 \
      --default-storage-class=STANDARD \
      --uniform-bucket-level-access
    ```

    If user provided an existing bucket name, skip this step.

    Set the bucket name variable for Step 3: `BUCKET_NAME=<confirmed-or-created-name>`

    **Step 2 — Enable Cloud SQL automated backups (SEC-04, per D-08):**

    ```bash
    gcloud sql instances patch bite \
      --project=ascent-web-260224-119 \
      --backup-start-time=02:00 \
      --retained-backups-count=7 \
      --enable-bin-log \
      --retained-transaction-log-days=7
    ```

    Wait for the operation to complete. This may take 1-2 minutes and may briefly restart the instance.

    Verify:
    ```bash
    gcloud sql instances describe bite \
      --project=ascent-web-260224-119 \
      --format="yaml(settings.backupConfiguration)"
    ```

    Expected output must contain:
    - `enabled: true`
    - `binaryLogEnabled: true`
    - `startTime: '02:00'`
    - `retainedBackups: 7`
    - `transactionLogRetentionDays: '7'`

    **Step 3 — Update Cloud Run env vars (per D-05, D-06):**

    Single command with all env vars (per D-05 — one revision, one health check):

    ```bash
    gcloud run services update bite-pos-demo \
      --region=us-central1 \
      --project=ascent-web-260224-119 \
      --update-env-vars="LOG_CHANNEL=stackdriver,FILESYSTEM_DISK=gcs,LIVEWIRE_TEMP_DISK=gcs,GCS_BUCKET=<BUCKET_NAME>,GOOGLE_CLOUD_PROJECT_ID=ascent-web-260224-119,SENTRY_LARAVEL_DSN=<REAL_DSN_FROM_TASK_1>"
    ```

    CRITICAL: Use `--update-env-vars` (additive), NOT `--set-env-vars` (destructive). The `--set-env-vars` flag would wipe DB_PASSWORD, APP_KEY, DB_SOCKET, and all other existing vars.

    **Step 4 — Verify Cloud Run env vars:**

    ```bash
    gcloud run services describe bite-pos-demo \
      --region=us-central1 \
      --project=ascent-web-260224-119 \
      --format="yaml(spec.template.spec.containers[0].env)"
    ```

    Confirm all 6 new env vars appear in the output alongside the existing ones (DB_PASSWORD, APP_KEY, DB_SOCKET, etc. should still be present).

    **Step 5 — Verify health endpoint:**

    Get the service URL and curl /health:
    ```bash
    SERVICE_URL=$(gcloud run services describe bite-pos-demo \
      --region=us-central1 \
      --project=ascent-web-260224-119 \
      --format="value(status.url)")
    curl -s "$SERVICE_URL/health" | python3 -m json.tool
    ```

    Expected: HTTP 200 with `"status": "healthy"` and all subsystem checks passing (db, storage, gd_webp, queue).
