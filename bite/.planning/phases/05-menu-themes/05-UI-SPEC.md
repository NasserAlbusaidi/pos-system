## Spacing Scale

This phase uses per-theme density — spacing varies intentionally by theme. The base 8-point scale applies everywhere except where a theme overrides it.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing within card body |
| sm | 8px | Default card body padding (Modern: compact), card gap (Modern: 8px grid gap) |
| md | 16px | Card gap (Dark: 16px grid gap), card padding (Dark: 16px internal) |
| lg | 24px | Section padding (page-level horizontal), category header vertical margin |
| xl | 32px | Theme picker card vertical padding |
| 2xl | 48px | Page top/bottom padding on guest menu |
| 3xl | 64px | Not used in guest-menu context |

### Per-Theme Spacing Contract (D-22–D-25)

| Token | Warm | Modern | Dark |
|-------|------|--------|------|
| `--theme-grid-gap` | 12px | 8px | 16px |
| `--theme-card-padding` | 10px | 8px | 16px |
| `--theme-image-height` | 140px | 80px | 200px |
| Category header vertical margin | 16px top / 12px bottom | 12px top / 8px bottom | 20px top / 16px bottom |

Exceptions:
- Touch targets (add-to-cart button, theme picker cards): minimum 44px in both dimensions — this overrides any spacing compression in Modern theme
- Sold-out badge position: 8px from card edge in all themes (consistent discoverability)
- RTL sold-out badge: left: 8px; letter-spacing: 0 (existing rule preserved across all themes)

Source: CONTEXT.md § Spacing Density (D-22–D-25), existing app.css patterns

## Typography

### Global (all themes)

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 | 1.5 | theme body font (see per-theme table) |
| Label / Price | 12px | 600 | 1.3 | JetBrains Mono (monospace — unchanged across themes) |
| Card Name | 14px | 600 | 1.3 | theme body font |
| Category Header | 20px | 700 | 1.2 | theme display font |

Exactly 3 sizes declared: 12px, 14px, 20px. Exactly 2 weights: 400 (regular) + 600–700 (semibold/bold).

### Per-Theme Font Application (D-30–D-34)

| Theme | `--theme-body-font` | `--theme-display-font` | CSS variable values |
|-------|---------------------|------------------------|---------------------|
| Warm | `'Rubik', system-ui, sans-serif` | `'Playfair Display', Georgia, serif` | `--theme-body-font: 'Rubik', system-ui, sans-serif` |
| Modern | `'Inter', system-ui, sans-serif` | `'Inter', system-ui, sans-serif` | `--theme-body-font: 'Inter', system-ui, sans-serif` |
| Dark | `'DM Sans', system-ui, sans-serif` | `'DM Serif Display', Georgia, serif` | `--theme-body-font: 'DM Sans', system-ui, sans-serif` |

### Letter-Spacing Rules

- Warm + Modern: `letter-spacing: 0.01em` on category headers (LTR only, scoped via [dir="ltr"])
- Dark: `letter-spacing: 0` on category headers (premium minimal look)
- RTL ([dir="rtl"]): `letter-spacing: 0` enforced globally for all themes — Arabic script requires no tracking
- Sold-out badge: `letter-spacing: 0.14em` LTR; `letter-spacing: 0` RTL (existing rule preserved)

Source: CONTEXT.md § Font Pairings (D-30–D-34), RESEARCH.md § Summary, STATE.md § Decisions ("letter-spacing must be scoped to [dir='ltr']")

## Color

### How the Color System Works

Brand colors (`--paper`, `--ink`, `--crema`) are owned by the branding cascade in the guest layout's inline `<style>` block. Themes MUST NOT overwrite these three tokens at runtime if a shop has custom branding. Themes DO set default values for these three tokens as fallbacks (via [data-theme] selector in app.css), which are overridden by the inline style when shop branding exists.

Themes own structural tokens and derived color defaults only (D-35, D-37, D-38).

### Per-Theme Default Palette (applies when shop has no custom brand colors — D-38)

| Token | Warm default | Modern default | Dark default |
|-------|-------------|----------------|--------------|
| `--paper` (dominant 60%) | `245 240 230` (warm cream) | `255 255 255` (pure white) | `14 14 18` (near-black) |
| `--ink` (text) | `44 37 32` (dark brown) | `15 15 15` (near-black) | `240 238 234` (off-white) |
| `--crema` (accent 10%) | `196 151 90` (warm gold) | `79 70 229` (indigo) | `200 160 80` (amber gold) |

### Accent (10%) — Reserved Elements

Accent (`--crema`) is reserved ONLY for:
- Add-to-cart button background (`.menu-product-add`)
- Active/selected state on cart item quantity badges
- Theme picker selected-state highlight border/ring
- "Save" CTA button background in theme picker settings panel

Accent is NOT used on: category headers, card borders, text labels, price tags, sold-out badges, or navigation.

### Semantic Colors (shared across themes)

| Role | Value | Usage |
|------|-------|-------|
| Destructive | `--alert: 186 59 48` | Destructive actions only (no destructive actions exist in guest menu; used in admin only) |
| Signal / Success | `--signal: 33 138 111` | Not used in Phase 5 guest menu |

### Card Styling per Theme (D-05–D-08)

| Property | Warm | Modern | Dark |
|----------|------|--------|------|
| `--theme-card-radius` | 12px | 0px | 8px |
| `--theme-card-shadow` | `0 4px 20px -6px rgb(0 0 0 / 0.15)` | `none` | `0 0 24px -8px rgb(var(--crema) / 0.25)` (glow) |
| `--theme-card-border` | `1px solid rgb(var(--line) / 0.8)` | `1px solid rgb(var(--line))` | `none` |

### Image Treatment per Theme (D-09–D-12)

| Theme | Image Treatment | Implementation |
|-------|-----------------|----------------|
| Warm | Soft rounded corners on image area (border-radius: 8px top-left/top-right); object-fit: contain (existing Sourdough requirement) | CSS on .menu-product-image-area per theme selector |
| Modern | Square crop, clean edges (border-radius: 0); object-fit: cover; no overlay | CSS on .menu-product-image-area [data-theme="modern"] |
| Dark | object-fit: cover + `::after` pseudo-element gradient overlay: `linear-gradient(180deg, transparent 40%, rgb(0 0 0 / 0.5) 100%)` | CSS on .menu-product-image-area [data-theme="dark"] |

Source: CONTEXT.md § Image Treatment (D-09–D-12), CONTEXT.md § Brand Color Interaction (D-35–D-38)

## Layout Structures

Three structurally distinct layouts — implemented with Blade `@if($theme === 'X')` conditionals where HTML differs, and CSS custom property values where only spacing/sizing differs (D-13–D-16).

### Warm: 2-Column Grid (D-14)

```
[Card] [Card]
[Card] [Card]
```

- `--theme-grid-cols: repeat(2, 1fr)`
- Category header: centered, Playfair Display, decorative underline via `::after` pseudo-element (2px, accent color, centered) — D-27
- Image area: 140px height, rounded top corners, object-fit: contain
- Cards: 12px gap, soft shadow

### Modern: Single-Column Horizontal List (D-15)

```
[Image | Name + Description + Price]
[Image | Name + Description + Price]
```

- `--theme-grid-cols: 1fr` — full-width single column
- Card HTML structure changes: flex-row, image fixed at 80px × 80px on left, text block on right
- Category header: left-aligned, Inter 600, 1px solid `--line` separator beneath — D-28
- Gap: 8px between cards (compact scanning)

### Dark: Full-Width Hero Cards (D-16)

```
[Full-width card with large image + overlay text]
[Full-width card with large image + overlay text]
```

- `--theme-grid-cols: 1fr` — single column, full width
- Image: 200px height, overlay gradient, product name + price rendered over image in overlay
- Category header: full-width, centered or left per RTL, `::before`/`::after` horizontal lines with subtle glow via `box-shadow` — D-29
- Gap: 16px between cards (luxurious feel)
- Card padding: 16px internal

### Responsive Breakpoints (Claude's discretion — D-17)

- Mobile first (default): all theme layouts as described above (single or 2-column as defined)
- `@media (min-width: 640px)`: Warm 2-col remains 2-col; Modern list increases image to 96px; Dark hero cards cap at 560px max-width centered
- `@media (min-width: 768px)`: Warm upgrades to 3-column grid; Modern list widens to 640px max-width; Dark hero cards remain single-column with max-width 640px
- Guest menu is a mobile-first context — breakpoints are enhancements, not the primary design target

## Theme Picker UI Contract

### Placement in Shop Settings (D-42 — Claude's discretion)

Theme picker appears as the first section in the Appearance tab of shop settings, before the brand color pickers. Rationale: theme is the structural foundation that brand colors layer on top of — setting theme first makes the color override relationship obvious.

### Picker Layout (D-39–D-41)

Three stacked vertical cards, full-width of the settings panel. Each card contains:
- Left: static mockup preview area (180px × 120px, styled inline with theme CSS, showing 2 fake product tiles in theme's style)
- Right: theme name (semibold, 16px) + 1-line description (14px, ink-soft)
- Selected state: 2px solid `--crema` border on entire card + checkmark icon (Unicode ✓ or inline SVG) in top-right corner
- Unselected state: 1px solid `--line` border

### Static Mockup Fake Content (D-40)

Each preview uses the same 2 fake products:
- "Croissant" — OMR 0.800
- "Cappuccino" — OMR 1.200

Mockup is pure HTML/CSS inline within the Blade template, scoped to the picker card. No PHP rendering, no image loading, no external requests.

### Live Preview (THEME-05)

- Alpine.js `x-data="{ previewTheme: '{{ $theme }}' }"` on the settings form wrapper
- Each theme picker card: `x-on:click="previewTheme = 'warm'"` (or 'modern'/'dark')
- Preview container: `x-bind:data-theme="previewTheme"` — a `<div>` below the picker showing a larger live mockup that inherits the data-theme tokens
- No Livewire round-trip for preview — Alpine.js only
- Selected state ring: `x-bind:class="previewTheme === 'warm' ? 'ring-2 ring-[rgb(var(--crema))]' : ''"` (or equivalent CSS class toggle)

## RTL Compatibility Contract (THEME-04, STATE.md hard requirement)

All three themes must satisfy:

| Rule | Detail |
|------|--------|
| `letter-spacing: 0` in RTL | Applied via `[dir="rtl"] *` or scoped per component — Arabic script must never have tracking |
| Font fallback | All three themes: `[dir="rtl"]` overrides body font to `'IBM Plex Sans Arabic', sans-serif` |
| Category header alignment | RTL: right-aligned for Warm and Modern; Dark: centered remains centered |
| Warm decorative underline | `[dir="rtl"]` centers underline identically (same `::after` behavior, RTL text context handles alignment) |
| Modern separator | Direction-agnostic `border-bottom` — works in both directions |
| Dark glow divider | Symmetric — direction-agnostic |
| Horizontal card (Modern) | `[dir="rtl"]` reverses flex direction to `row-reverse` so image appears on right |
| Sold-out badge | Existing `[dir="rtl"] .menu-product-sold-out-badge { left: 8px; letter-spacing: 0 }` preserved as-is |
| Screenshot required | At least one RTL screenshot of each theme is required before merge (STATE.md blocker note) |

## CSS Architecture Contract

### Token Naming Convention

All new theme-specific custom properties use the `--theme-` prefix:

```css
/* Structural (always apply — not overridable by branding) */
--theme-card-radius
--theme-card-shadow
--theme-card-border
--theme-grid-cols
--theme-grid-gap
--theme-image-height
--theme-body-font
--theme-display-font
--theme-card-padding

/* Default palette (overridable by branding inline style) */
/* Uses existing --paper / --ink / --crema names */
/* Set inside [data-theme="X"] {} block in app.css */
```

### Cascade Order (lowest to highest specificity wins last)

1. `:root {}` — base tokens in app.css (existing)
2. `[data-theme="X"] {}` — theme defaults (new, in app.css)
3. Inline `<style>` in guest layout — shop brand colors override `--paper`, `--ink`, `--crema` (existing mechanism)

This cascade satisfies D-35 and D-36 without any JavaScript logic.

### Placement in app.css

New theme blocks go under `@layer components` after existing `.menu-*` class definitions. Do not place in `:root` — they must be attribute-scoped.
