<objective>
Rewrite the Docker container from a demo-grade PHP CLI setup to a production-grade Nginx + PHP-FPM architecture, configure Cloud SQL MySQL 8.0 connectivity via Unix socket, and enforce that no secrets are baked into the container image.

Purpose: The current container uses `php artisan serve` (single-threaded, no concurrent requests) with SQLite. Production needs Nginx for static assets and concurrent PHP handling via FPM, MySQL for a real database, and environment-based secrets for Cloud Run deployment.

Output: A `docker build` that produces a container running Nginx + PHP-FPM via supervisord, configured for Cloud SQL Auth Proxy, with all secrets injected at runtime.
</objective>

    - `error_log /dev/stderr;` and `access_log /dev/stdout;` (Cloud Run captures stdout/stderr)
    - Single `server` block listening on port 8080 (Cloud Run expects PORT env var, default 8080)
    - `root /var/www/html/public;` (Laravel's public directory)

    - `[www]` pool section

    - `listen.owner = www-data` and `listen.group = www-data`
    - `pm = dynamic`
    - `pm.max_children = 10`
    - `pm.start_servers = 2`
    - `pm.min_spare_servers = 1`
    - `pm.max_spare_servers = 4`
    - `upload_max_filesize = 20M`
    - `post_max_size = 50M`
    - `max_file_uploads = 10`
    - `memory_limit = 256M`
    - `clear_env = no` (CRITICAL: Cloud Run injects env vars into the container; FPM must not clear them)

    **docker/supervisord.conf** — Process manager for single-container multi-process:

    ```dockerfile
    FROM php:8.4-fpm-bookworm AS app

    WORKDIR /var/www/html

    # Install system deps: nginx, supervisord, MySQL client libs, GD libs (for intervention/image)
    RUN apt-get update \
        && apt-get install -y --no-install-recommends \
            nginx \
            supervisor \
            libfreetype6-dev \
            libjpeg62-turbo-dev \
            libpng-dev \
            libwebp-dev \
            unzip \
        && docker-php-ext-configure gd --with-freetype --with-jpeg --with-webp \
        && docker-php-ext-install pdo pdo_mysql bcmath gd \
        && rm -rf /var/lib/apt/lists/*

    COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

    # Copy configs
    COPY docker/nginx.conf /etc/nginx/nginx.conf
    COPY docker/php-fpm.conf /usr/local/etc/php-fpm.d/www.conf
    COPY docker/supervisord.conf /etc/supervisor/conf.d/supervisord.conf

    # Copy app source + frontend build artifacts
    COPY . .
    COPY --from=frontend /app/public/build ./public/build

    # Production environment defaults (NO secrets — per D-10, D-11)
    ENV APP_ENV=production \
        APP_DEBUG=false \
        LOG_CHANNEL=stderr \
        CACHE_STORE=file \
        SESSION_DRIVER=file \
        QUEUE_CONNECTION=sync \
        MAIL_MAILER=log \
        DB_CONNECTION=mysql \
        SENTRY_ENVIRONMENT=production \
        SENTRY_TRACES_SAMPLE_RATE=0.1

    # Build-time key for artisan commands (NOT a real secret — per D-11)
    ARG BUILD_APP_KEY="base64:AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA="

    RUN APP_KEY="${BUILD_APP_KEY}" \
        composer install --prefer-dist --no-interaction --optimize-autoloader --no-scripts --no-dev \
        && mkdir -p storage/framework/cache storage/framework/sessions storage/framework/views storage/logs bootstrap/cache \
        && rm -f bootstrap/cache/packages.php bootstrap/cache/services.php \
        && APP_KEY="${BUILD_APP_KEY}" php artisan package:discover --ansi \
        && APP_KEY="${BUILD_APP_KEY}" php artisan optimize:clear \
        && chown -R www-data:www-data storage bootstrap/cache \
        && chmod +x artisan

    COPY docker/start.sh /usr/local/bin/start-laravel
    RUN chmod +x /usr/local/bin/start-laravel

    EXPOSE 8080

    CMD ["start-laravel"]
    ```
