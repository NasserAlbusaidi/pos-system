Migrate file storage from local disk to Google Cloud Storage so that product images and uploads are stored in a GCS bucket with public URLs, and Livewire temporary uploads work across multi-instance Cloud Run deployments.

Output: GCS filesystem driver configured, ImageService refactored for stream-based operations, productImage helper returning GCS URLs, Livewire temp upload disk set to GCS.
</objective>

    Refactor ImageService to use stream-based operations instead of absolute filesystem paths (per D-09):

    **app/Services/ImageService.php — processUpload() rewrite:**

    The current flow is:
    1. `Storage::disk($disk)->path($storedPath)` to get absolute path
    2. `$manager->read($absolutePath)` to load image from filesystem
    3. `file_put_contents($absolutePath, $encoded)` to save variant

    GCS doesn't support `->path()` (no local filesystem). The new flow must:
    1. `Storage::disk($disk)->get($storedPath)` to read file contents as a string
    2. `$manager->read($contents)` to load image from binary string (Intervention v3 supports this)
    3. `Storage::disk($disk)->put($variantPath, $encoded->toString())` to write via Storage abstraction

    Full replacement for `processUpload()`:
    ```php
    public function processUpload(string $storedPath, string $disk = 'public'): string
    {
        $contents = Storage::disk($disk)->get($storedPath);
        $ext = $this->supportsWebp() ? 'webp' : 'jpg';
        $baseName = preg_replace('/\.[^.]+$/', '', $storedPath);
        $manager = new ImageManager(new GdDriver);

        $variants = [
            'thumb' => 200,
            'card'  => 400,
            'full'  => 800,
        ];

        foreach ($variants as $variant => $size) {
            $image = $manager->read($contents);
            $image->scaleDown(width: $size, height: $size);

            $encoded = $this->supportsWebp()
                ? $image->toWebp(quality: 80)
                : $image->toJpeg(quality: 85);

            $variantPath = "{$baseName}-{$variant}.{$ext}";
            Storage::disk($disk)->put($variantPath, $encoded->toString());
        }

        // Only delete the original after ALL variants are saved
        Storage::disk($disk)->delete($storedPath);

        return "{$baseName}-full.{$ext}";
    }
    ```

    **Remove the `saveVariant()` method entirely** — it used `file_put_contents()` and `mkdir()` which are local filesystem operations. The Storage facade handles this now.

    **`deleteVariants()` method** — no changes needed. It already uses `Storage::disk($disk)->delete($paths)` which works with any driver.

    **`supportsWebp()` method** — no changes needed.

    **app/Helpers/image.php — productImage() rewrite:**

    The current helper hardcodes `/storage/` prefix which only works for local public disk. For GCS, the URL is `https://storage.googleapis.com/BUCKET/path`. The fix: use Laravel's `Storage::disk()->url()` method which returns the correct URL for any driver.

    Replace the entire function:
    ```php
    function productImage(?Product $product, string $variant = 'card'): ?string
    {
        if ($product === null || $product->image_url === null) {
            return null;
        }

        // image_url is e.g. "products/abc123-full.webp"
        // Replace "-full." with "-{$variant}."
        $variantPath = preg_replace('/-full\./', "-{$variant}.", $product->image_url);

        return Storage::disk(config('filesystems.default'))->url($variantPath);
    }
    ```

    Add `use Illuminate\Support\Facades\Storage;` at the top of the file (after the `use App\Models\Product;` line).

    This returns:
    - Local dev (public disk): `/storage/products/abc123-card.webp`
    - Production (gcs disk): `https://storage.googleapis.com/BUCKET/products/abc123-card.webp`

    **Important:** The `public` disk's `url` config already generates `/storage/...` URLs (via the `'url'` key in filesystems.php). The GCS driver generates `https://storage.googleapis.com/BUCKET/...` URLs automatically. So `Storage::disk()->url()` is the correct abstraction for both.
