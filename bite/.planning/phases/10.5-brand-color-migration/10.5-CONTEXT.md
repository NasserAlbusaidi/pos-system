<decisions>
## Implementation Decisions

### Token Remap Strategy

- **D-01: In-place value swap.** Legacy token names (`--paper`, `--ink`, `--crema`, `--canvas`, `--panel`, `--panel-muted`, `--line`, `--ink-soft`, `--signal`, `--alert`, `--focus`) keep their semantic role. Only their VALUES change. All ~85 component references (`rgb(var(--paper))`, `bg-paper`, `text-ink`, etc.) auto-inherit the new colors. The shop-branding inline `<style>` block in `app.blade.php` (lines 28–89) continues to override `--paper`/`--ink`/`--crema` (and derived neutrals) on tenant routes — override mechanism unchanged.
- **D-02: Forest hierarchy.** `--crema` rebinds to `#004225` (Bite Primary, the deep forest). `--ink` stays near-black for high-contrast text. Hover/active states use `--brand-accent-1` (`#37B34A`) — i.e., `.btn-primary:hover` background changes from `--crema` to `--brand-accent-1`.
- **D-03: `--paper` = `#FAFAF7`** (off-white, slightly warmer than pure white). Reads as "paper" not "screen"; pairs with deep-forest accents without going clinical.
- **D-04: Single brand-green for status + focus.** `--signal` rebinds to `#37B34A` (was `#218A6F` teal). `--focus` rebinds to `#37B34A` (was `#FA8148` orange). One vibrant green covers both success states and focus rings. `--alert` keeps its red (`#BA3B30`).
- **D-05: Lime accents reserved.** `--brand-accent-2: #7AC70C` and `--brand-accent-3: #B7C40D` are defined in `:root` for forward use (charts, badges, marketing splashes in v1.4+) but are **not** bound to any legacy token in v1.3 chrome.
- **D-06: Derived neutrals via linear RGB interpolation.** `--canvas`, `--panel`, `--panel-muted`, `--line`, `--ink-soft` are computed at `:root` level using the **same algorithm and mix ratios** as the existing shop-branding cascade in `app.blade.php` lines 56–72:
  ```
  --canvas      = mix(paper, ink, 0.06)   // barely darker than paper
  --panel       = mix(paper, white, 0.30) // slightly lighter than paper
  --panel-muted = mix(paper, ink, 0.12)   // skeleton shimmer bg
  --line        = mix(paper, ink, 0.18)   // border color
  --ink-soft    = mix(ink, paper, 0.55)   // secondary text
  ```
  With `--paper: #FAFAF7` and `--ink` near-black, these resolve to neutral grays. Same algorithm shop branding uses → consistent behavior across platform vs tenant.
- **D-07: `--brand-secondary` defined, reserved.** `--brand-secondary: #0B6B2E` lives in `:root` for Phase 12 themes (dark variant, hover hierarchies) but has no platform-chrome role in 10.5.

### Body Background

- **D-08: Drop the multi-stop body gradient.** The current `body { background: radial-gradient(...) , radial-gradient(...) , linear-gradient(...) }` rule (`app.css` ~line 175–179) becomes `body { background: rgb(var(--paper)); }`. The hardcoded `rgb(231 230 223)` literal at the end of the linear gradient is removed by this change.
- **D-09: Keep `body::before` noise dot pattern.** The 4px-grid radial-gradient texture stays — preserves subtle warmth, no color cost.
- **D-10: Apply globally.** The simpler body rule applies to every route (platform AND tenant). Tenant routes still feel right because the shop-branding inline `<style>` overrides `--paper` per shop — `/menu/sourdough` reads warm cream because Sourdough's `--paper` IS warm cream, not because the gradient interpolation says so.

### Admin Layout vs Shop Branding

- **D-11: Strip shop-branding injection from `admin.blade.php`.** Remove the `@if($shop) … @endif` block (lines 14–41). Every `/admin/*` route reads platform tokens (`#FAFAF7` paper, `#004225` crema, `#37B34A` accent-1, etc.). `/dashboard` and friends always show Bite green regardless of which shop the user belongs to.
- **D-12: Scoped preview on `/admin/shop-settings`.** Wrap the color picker preview swatch / sample card in a self-contained `<div>` with its own inline `<style>` that applies the user-entered hex values **scoped to that container only** (e.g., `#shop-branding-preview { --paper: ...; --ink: ...; --crema: ...; }`). Owner sees their colors in the preview, not on the whole admin page. Specific UI shape (swatch row vs mini-menu vs modal) is Claude's discretion.
- **D-13: `app.blade.php` injection unchanged.** The shop-branding `<style>` block on lines 28–89 stays exactly as-is. `/menu/*`, `/pos/*`, `/kds`, `/track/*` keep their per-shop cascade. POS is tenant-facing per the locked v1.3 decision (`STATE.md`).
- **D-14: `super-admin.blade.php` and `guest.blade.php` unchanged for branding.** Neither layout currently injects shop branding; no change. (Hardcoded color leaks in `guest.blade.php` are addressed under D-17.)
- **D-15: Plan 10.5-02 regression test = Laravel feature test.** Assert that GET `/menu/{slug}` for a shop with custom branding returns HTML containing a `<style>` block with the shop's `--paper` RGB triple (Sourdough: `255 228 214`). Add a counter-assertion: GET `/super-admin/shops` (as super_admin) returns HTML where computed `--paper` resolves to `250 250 247` (the platform default — RGB triple of `#FAFAF7` per D-03). Lives in `tests/Feature/BrandingCascadeTest.php`. SQLite in-memory + sync queue (per phpunit.xml).

<specifics>
## Specific Ideas

- "Forest hierarchy" — the brand reads as a real green identity, not a green accent on top of warm cream. `--paper` = off-white, `--crema` = deep forest, hover = vibrant green. Bite Primary (`#004225`) is the dominant chrome color, the way the warm orange used to be.
- Single brand-green for `--signal` and `--focus` keeps the mental model simple: "green = positive system state." Don't introduce two greens fighting for attention.
- Reserve `#7AC70C` and `#B7C40D` rather than forcing roles. The brand pack supplied them as a palette, not as states. Charts, badges, and v1.4 dark-mode are natural homes once those phases land.
- Preserve the multi-tenant white-label promise: shops like Sourdough must keep their own warm cream/brown on customer-facing screens. The shop-branding inline `<style>` cascade in `app.blade.php` is the contract. Phase 10.5 must not weaken it.
- Drop the multi-stop body gradient — modern SaaS look (Linear / Vercel). Solid surface lets green-accented panels pop without color noise underneath. Keep the `body::before` noise dot pattern as the only texture; it's color-agnostic.
- Strip shop-branding injection from `admin.blade.php` so admins (the operational shell) read Bite green even when an owner is logged in to their own shop. Live preview on `/admin/shop-settings` becomes a scoped widget instead of taking over the whole layout.

</specifics>
