## Color

The Bite color system has three layers — brand source of truth, platform chrome, and per-shop white-label.

### Brand tokens

Defined in `resources/css/app.css :root`:

| Token | Value | Role |
|-------|-------|------|
| `--brand-primary`   | `#004225` | 60% — dominant chrome (deep forest) |
| `--brand-secondary` | `#0B6B2E` | reserved (dark theme, hover hierarchies — Phase 12) |
| `--brand-accent-1`  | `#37B34A` | 30% — hover, focus, signal/success |
| `--brand-accent-2`  | `#7AC70C` | reserved (charts, badges) |
| `--brand-accent-3`  | `#B7C40D` | reserved (lime accents, marketing) |

Tokens are stored as space-separated RGB triples (e.g. `--brand-primary: 0 66 37`) so callers can compose alpha via `rgb(var(--brand-primary) / 0.5)`.

### Three layers

1. **Brand tokens** (source of truth) — the table above.
2. **Platform tokens** (legacy names, brand-derived values) — `--paper`, `--ink`, `--crema`, `--canvas`, `--panel`, `--panel-muted`, `--line`, `--ink-soft`, `--signal`, `--alert`, `--focus`. Names kept for code stability; values rebound to the forest hierarchy. Platform chrome (super-admin, admin, billing, login, welcome) reads these.
3. **Per-shop tokens** (multi-tenant override) — `Shop::branding` JSON drives an inline `<style>` block in `resources/views/layouts/app.blade.php` that overrides `--paper` / `--ink` / `--crema` and recomputes derived neutrals via linear RGB interpolation. Tenant routes (`/menu/*`, POS, KDS, receipts, order tracking) read these.

### Where the boundary lives

- `resources/views/layouts/app.blade.php` — injects shop branding (tenant-facing).
- `resources/views/layouts/admin.blade.php` — no injection (platform chrome, always Bite green).
- `resources/views/layouts/super-admin.blade.php`, `resources/views/layouts/guest.blade.php` — no injection.
- `resources/css/app.css :root` — brand source of truth.

### 60/30/10 ratio

- ~60% `--brand-primary` — background fills, large surfaces, primary CTAs.
- ~30% `--brand-accent-1` — hover states, focus rings, success affordances.
- ~10% `--brand-accent-2` / `--brand-accent-3` / `--alert` — warnings, badges, charts, lime marketing splashes.

### Picking the right token

✅ Do
```css
.cta { background: rgb(var(--crema)); }
/* platform chrome — auto-resolves Bite green;
   on tenant routes the inline <style> override rebinds --crema to the shop's accent.
   White-label preserved. */

.cta:hover { background: rgb(var(--brand-accent-1)); }
/* explicit brand hover — identical across platform and tenant. */
```

❌ Don't
```css
.cta { background: #004225; }
/* hardcoded — bypasses tenant override. Sourdough's customer-facing menu
   would show Bite green instead of warm cream. */

.cta { background: rgb(var(--brand-primary)); }
/* on a tenant-facing surface — kills white-label.
   Use --crema (or the platform legacy token that rebinds per shop). */
```
```
