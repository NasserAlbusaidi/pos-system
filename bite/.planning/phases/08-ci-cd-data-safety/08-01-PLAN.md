Add the `deploy` job to `.github/workflows/ci.yml` after the existing `test` job. This implements DEPLOY-04 and decisions D-01, D-02, D-05, D-06, D-12, D-14, D-15.

The deploy job must have these exact properties:
- **name:** `Build & Deploy to Cloud Run`
- **runs-on:** `ubuntu-latest`
- **needs:** `[test]` — test job must pass first (D-01: failed tests prevent deployment)
- **if:** `github.ref == 'refs/heads/main' && github.event_name == 'push'` — only on push to main, not PRs (D-02)
- **permissions:** `contents: read` and `id-token: write` (required for WIF OIDC token)

Steps in order:

1. **Checkout** — `actions/checkout@v4`

2. **WIF Authentication** — `google-github-actions/auth@v3` with:
   - `workload_identity_provider: ${{ secrets.WIF_PROVIDER }}`
   - `service_account: ${{ secrets.WIF_SERVICE_ACCOUNT }}`

3. **Setup gcloud** — `google-github-actions/setup-gcloud@v2`

4. **Capture current revision for rollback** (id: `prev-revision`) — BEFORE deploy (Pitfall 2 from RESEARCH.md):
```bash
REV=$(gcloud run services describe $SERVICE \
  --region=$REGION --project=$PROJECT_ID \
  --format="value(status.traffic[0].revisionName)" 2>/dev/null || echo "")
echo "name=$REV" >> "$GITHUB_OUTPUT"
```

5. **Setup Docker Buildx** — `docker/setup-buildx-action@v3` (required for GHA cache backend)

6. **Configure Docker auth for Artifact Registry**:
```bash
gcloud auth configure-docker us-central1-docker.pkg.dev --quiet
```

7. **Build and push Docker image** — `docker/build-push-action@v6` with:
   - `context: .`
   - `push: true`
   - Tags (per D-12 — SHA + latest):
     ```
     ${{ env.IMAGE_PATH }}:${{ github.sha }}
     ${{ env.IMAGE_PATH }}:latest
     ```
   - `cache-from: type=gha`
   - `cache-to: type=gha,mode=max`

8. **Deploy to Cloud Run** (id: `deploy`) — `google-github-actions/deploy-cloudrun@v3` with:
   - `service: ${{ env.SERVICE }}`
   - `image: ${{ env.IMAGE_PATH }}:${{ github.sha }}`
   - `region: ${{ env.REGION }}`

9. **Wait for container startup** — `sleep 30` (per D-05: wait 30 seconds for migrations + cache warming in start.sh)

10. **Post-deploy health check** (id: `health-check`) — curl with 3 retries, 10s backoff:
```bash
HEALTH_URL="${{ steps.deploy.outputs.url }}/health"
for i in 1 2 3; do
  STATUS=$(curl -s -o /dev/null -w "%{http_code}" "$HEALTH_URL")
  echo "Attempt $i: HTTP $STATUS"
  if [ "$STATUS" = "200" ]; then
    echo "Health check passed"
    exit 0
  fi
  [ $i -lt 3 ] && sleep 10
done
echo "Health check failed after 3 attempts"
exit 1
```

11. **Rollback on health check failure** (per D-06) — only runs `if: failure() && steps.prev-revision.outputs.name != ''`:
```bash
echo "Rolling back to ${{ steps.prev-revision.outputs.name }}"
gcloud run services update-traffic $SERVICE \
  --region=$REGION \
  --project=$PROJECT_ID \
  --to-revisions="${{ steps.prev-revision.outputs.name }}=100"
```

Also add a **Docker build validation step** to the `test` job (per D-02) — after lint, add:
```yaml
- name: Validate Docker build
  if: github.event_name == 'pull_request'
  run: docker build --target app -t bite-pos-test .
```
This catches Dockerfile and asset build failures on PRs without pushing anywhere.
