The CI/CD workflow requires GCP authentication via Workload Identity Federation. Run these gcloud commands from your local machine (you must be authenticated as a GCP project owner).

**Step 1: Create Workload Identity Pool**
```bash
gcloud iam workload-identity-pools create "github-actions" \
  --project="ascent-web-260224-119" \
  --location="global" \
  --display-name="GitHub Actions"
```

**Step 2: Create OIDC Provider** (scoped to your repo: NasserAlbusaidi/pos-system)
```bash
gcloud iam workload-identity-pools providers create-oidc "github-repo" \
  --project="ascent-web-260224-119" \
  --location="global" \
  --workload-identity-pool="github-actions" \
  --issuer-uri="https://token.actions.githubusercontent.com" \
  --attribute-mapping="google.subject=assertion.sub,attribute.repository=assertion.repository" \
  --attribute-condition="assertion.repository == 'NasserAlbusaidi/pos-system'"
```

**Step 3: Create CI Service Account**
```bash
gcloud iam service-accounts create "github-actions-ci" \
  --project="ascent-web-260224-119" \
  --display-name="GitHub Actions CI"
```

**Step 4: Grant required IAM roles**
```bash
gcloud projects add-iam-policy-binding ascent-web-260224-119 \
  --member="serviceAccount:github-actions-ci@ascent-web-260224-119.iam.gserviceaccount.com" \
  --role="roles/artifactregistry.writer"

gcloud projects add-iam-policy-binding ascent-web-260224-119 \
  --member="serviceAccount:github-actions-ci@ascent-web-260224-119.iam.gserviceaccount.com" \
  --role="roles/run.admin"

gcloud projects add-iam-policy-binding ascent-web-260224-119 \
  --member="serviceAccount:github-actions-ci@ascent-web-260224-119.iam.gserviceaccount.com" \
  --role="roles/iam.serviceAccountUser"
```

**Step 5: Allow WIF pool to impersonate the service account**

First, get the project number (should be 528372920943 per research):
```bash
gcloud projects describe ascent-web-260224-119 --format="value(projectNumber)"
```

Then bind:
```bash
gcloud iam service-accounts add-iam-policy-binding \
  "github-actions-ci@ascent-web-260224-119.iam.gserviceaccount.com" \
  --project="ascent-web-260224-119" \
  --role="roles/iam.workloadIdentityUser" \
  --member="principalSet://iam.googleapis.com/projects/528372920943/locations/global/workloadIdentityPools/github-actions/attribute.repository/NasserAlbusaidi/pos-system"
```

**Step 6: Get WIF_PROVIDER value**
```bash
gcloud iam workload-identity-pools providers describe "github-repo" \
  --project="ascent-web-260224-119" \
  --location="global" \
  --workload-identity-pool="github-actions" \
  --format="value(name)"
```
This outputs the full resource name (e.g., `projects/528372920943/locations/global/workloadIdentityPools/github-actions/providers/github-repo`).

**Step 7: Add GitHub Actions secrets**

Go to https://github.com/NasserAlbusaidi/pos-system/settings/secrets/actions and add:

| Secret Name | Value |
|-------------|-------|
| `WIF_PROVIDER` | Full resource name from Step 6 |
| `WIF_SERVICE_ACCOUNT` | `github-actions-ci@ascent-web-260224-119.iam.gserviceaccount.com` |
| `GCP_PROJECT_ID` | `ascent-web-260224-119` |
| `CLOUD_RUN_SERVICE` | `bite-pos-demo` |
| `GCP_REGION` | `us-central1` |

**Step 8: Wait 5 minutes** for WIF propagation (Pitfall 7 from research).

  <action>
Configure Cloud SQL automated backups with 7-day retention and PITR. Run these gcloud commands from your local machine.

**Step 1: Enable Cloud SQL Admin API** (currently disabled — Pitfall 3)
```bash
gcloud services enable sqladmin.googleapis.com --project=ascent-web-260224-119
```
Wait 2-5 minutes for API propagation.

**Step 2: Find your Cloud SQL instance name**

Option A — from GCP Console: Go to https://console.cloud.google.com/sql/instances?project=ascent-web-260224-119

Option B — from Cloud Run env: Check the `DB_SOCKET` environment variable on the `bite-pos-demo` service. It has the format `/cloudsql/ascent-web-260224-119:us-central1:INSTANCE_NAME`. The instance name is the last segment.

Option C — via gcloud (after API is enabled):
```bash
gcloud sql instances list --project=ascent-web-260224-119
```

**Step 3: Enable backups + PITR** (replace INSTANCE_NAME with actual name from Step 2)
```bash
gcloud sql instances patch INSTANCE_NAME \
  --project=ascent-web-260224-119 \
  --backup-start-time=02:00 \
  --retained-backups-count=7 \
  --enable-bin-log \
  --retained-transaction-log-days=7
```

This configures:
- Daily automated backups at 02:00 UTC (6 AM GST, before Oman business hours) per D-09
- 7 retained backups (7-day retention) per D-09
- Binary logging enabled for PITR per D-10
- 7-day transaction log retention for PITR per D-10

**Step 4: Verify backup configuration**
```bash
gcloud sql instances describe INSTANCE_NAME \
  --project=ascent-web-260224-119 \
  --format="yaml(settings.backupConfiguration)"
```

Expected output:
```yaml
settings:
  backupConfiguration:
    backupRetentionSettings:
      retainedBackups: 7
      retentionUnit: COUNT
    binaryLogEnabled: true
    enabled: true
    startTime: 02:00
    transactionLogRetentionDays: 7
```

**For future reference — PITR restore command** (restores to a clone, original stays intact):

  --point-in-time '2026-03-27T03:00:00Z' \
  --project=ascent-web-260224-119
