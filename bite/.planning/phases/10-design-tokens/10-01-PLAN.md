    ```css
            /* Typography scale — see docs/design-system.md (added in Phase 10 / DS-03).
               Rounded whole-px values for crisp rendering on POS hardware (D-06).
               Base 16px (D-05); 7 t-shirt sizes (D-01). */

            --font-weight-regular: 400;
            --font-weight-medium: 500;
            --font-weight-semibold: 600;
            --font-weight-bold: 700;

            /* Line-heights — Latin track (Rubik). Arabic override block follows :root.
               D-08: two-track system; Arabic gets +0.15 leading for IBM Plex glyph metrics. */
            --line-height-tight: 1.20;
            --line-height-normal: 1.50;
            --line-height-loose: 1.70;
    ```

    Token order is grouped by purpose (sizes → weights → line-heights), not alphabetical, because grouping reads cleanest when the next phase opens this file (Claude's discretion per CONTEXT.md).

    ```css

        /* Arabic line-height override (D-08) — IBM Plex Sans Arabic ascenders are taller
           than Rubik, so Arabic text needs +0.15 leading at every level to avoid feeling cramped. */
        [lang="ar"] {
            --line-height-tight: 1.35;
            --line-height-normal: 1.65;
            --line-height-loose: 1.85;
        }
    ```

    The block sits inside `@layer base` (still inside the `@layer base { ... }` wrapper that opens at line 152 and closes much later — verify by checking that the `}` matching the `@layer base {` opener is below the components layer opener).
