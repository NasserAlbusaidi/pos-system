## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps, inline padding within card name/price row |
| sm | 8px | Gap between cards in the 2-column grid; card body padding (horizontal) |
| md | 16px | Section horizontal padding; category header left padding (accent bar clearance) |
| lg | 24px | Category header vertical padding above/below |
| xl | 32px | Bottom bar internal padding; section bottom margin |
| 2xl | 48px | Page bottom clearance (so bottom bar does not obscure last card) |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Touch targets for `+` quick-add button: minimum 44×44px (mobile tap target accessibility standard). Button visual size can be smaller (32px circle); add `min-width: 44px; min-height: 44px` with negative margin compensation.
- Image container height: fixed at 120px (`min-height: 120px`) for consistent card height regardless of image presence (GMVIZ-10).

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Rubik | 14px | 400 (regular) | 1.5 | Product description text (hidden by default, revealed on expand) |
| Label | Rubik | 12px | 600 (semibold) | 1.4 | Price display; `+` button label; meta labels |
| Product Name | Rubik | 14px | 600 (semibold) | 1.3 | Compact card product name — sentence case, not uppercase |
| Category Header | Playfair Display | 20px | 700 (bold) | 1.2 | Category section headings — serif, warm artisan feel |

**Weight constraint:** Exactly 2 Rubik weights in use — 400 (regular) and 600 (semibold). Category headers use Playfair Display 700, which is a separate display-only font and is NOT counted against the Rubik weight budget. Effective palette: Rubik 400, Rubik 600, Playfair Display 700.

**Sentence case rule (D-12):** Product names must NOT use CSS `text-transform: uppercase`. Remove the existing `uppercase` Tailwind class from the `<h4>`. Names render as-stored from the database.

### Derived Token Algorithm (D-17 — Claude's discretion, now specified)

All derived tokens are computed in `layouts/app.blade.php` using linear RGB interpolation:

| Token | Derivation | Effect |
|-------|------------|--------|
| `--canvas` | mix(paper, ink, 6%) | Page background — paper darkened barely; the 60% surface |
| `--panel` | mix(paper, white, 30%) | Card surface — paper lightened; the 30% secondary surface |
| `--panel-muted` | mix(paper, ink, 12%) | Muted card areas, skeleton shimmer base |
| `--line` | mix(paper, ink, 18%) | Borders — warm mid-tone, not cold grey |
| `--ink-soft` | mix(ink, paper, 55%) | Secondary text — dark brown softened toward cream |

### 60/30/10 Split

| Role | Value | Specific Elements |
|------|-------|-------------------|
| Dominant (60%) | `--canvas` (derived from paper) | Page background gradient (`body` background); the warm cream backdrop |
| Secondary (30%) | `--panel` (derived from paper lightened) | Product card surfaces (`.surface-card`); sticky header `bg-panel/85`; bottom bar background |
| Accent (10%) | `--crema` (gold, `#C4975A`) | Reserved for: `+` quick-add button background, cart review bottom bar, "Flash Sale" badge, language toggle active state, focus ring on interactive elements |

**Accent constraint (D-19):** Accent gold is NOT used on card backgrounds, card borders, category headers, or description text. Cards stay neutral paper/panel tones. Accent appears only on action-triggering elements.

**Primary visual anchor:** The `+` quick-add button is the primary visual anchor on each card. The accent gold circle (`--crema` background) against the neutral `--panel` card surface draws the eye directly to the primary action. No other element on the card uses accent color, ensuring the `+` button reads as the singular call to action at a glance.

### Semantic Colors

| Role | Value | Usage |
|------|-------|-------|
| Destructive | `--alert` (186 59 48) | No destructive actions in this phase — not used |
| Signal | `--signal` (33 138 111) | Not used in guest menu; admin-side only |

### Expand/Collapse Animation (D-03 — Claude's discretion, now specified)

Use Alpine `x-collapse` plugin (bundled with Livewire 3's Alpine distribution). Apply `x-collapse` directive to the description `<div>`. If `x-collapse` is unavailable, fall back to CSS `max-height` transition:

```css
.menu-product-description {
    max-height: 0;

}
```

Animation duration: 280ms. Easing: `cubic-bezier(0.23, 1, 0.32, 1)` (matches existing project animation curves — see `.fade-rise` in app.css).

## Interaction Contract

### Card Grid (GMVIZ-04/05)

- Layout: `grid-template-columns: repeat(2, 1fr)` on ALL screen sizes (no responsive breakpoints — D-01)
- Gap: 8px between cards
- No horizontal scroll; grid fills container width

### Card Expand/Collapse (D-03/D-04)

- State: Alpine `x-data="{ expanded: null }"` on the grid wrapper
- Trigger: Tap anywhere on card except `+` button
- Accordion: `expanded = (expanded === productId) ? null : productId` — one card open at a time (D-04)
- `+` button uses `wire:click.stop` to prevent card expand on quick-add tap (D-05)
- Collapsed state: shows image + name + price only
- Expanded state: description text revealed below price row, card height grows in-place, other cards in same column shift down
- No expanded-state add-to-cart button — CTA is always the `+` icon on compact card (D-06)

### Quick-Add Button (D-05)

- Position: bottom-right of card body area, aligned with price
- Tap target: 44×44px minimum (visual circle 28px, tap area extended via negative margin)
- Color: accent gold (`--crema`) background, white (`--panel`) plus sign
- On tap: `wire:click.stop="addToCart($productId)"` — fire and forget, no loading state on button
- If product has modifiers: wire:click triggers modifier modal (existing pattern from GuestMenu.php — no change needed)

### Skeleton Loading (GMVIZ-07)

- Image area shows `.skeleton` shimmer (left-to-right, 1.5s, warm tones) while `!loaded && !broken`
- On `@load`: `loaded = true` — shimmer hidden, image shown (`x-show="loaded && !broken"`)
- On `@error`: `broken = true` — shimmer hidden, placeholder SVG shown

### Background Gradient (D-18)

Replace the existing admin-facing body gradient with a guest-menu-specific warm gradient derived from branding tokens:

```css
/* Guest menu body override — applied via inline <style> in app.blade.php */
body {
    background: linear-gradient(
        180deg,
        rgb(var(--paper)) 0%,
        rgb(var(--canvas)) 100%
    );
    min-height: 100vh;
}
```

Subtle top-to-bottom fade: pure paper at top, slightly deeper canvas at bottom. No radial accents for guest menu (those are for admin dashboard).

## Branding Cascade Contract

### Token Derivation in `layouts/app.blade.php`

The `@php` block must compute and the `<style>` block must emit all 6 derived tokens:

```php
// Derivation (RGB linear interpolation)
--canvas:      mix(paper, ink, 0.06)   // barely darker than paper
--panel:       mix(paper, white, 0.30) // slightly lighter than paper
--panel-muted: mix(paper, ink, 0.12)   // skeleton shimmer bg
--line:        mix(paper, ink, 0.18)   // warm border color
--ink-soft:    mix(ink, paper, 0.55)   // secondary text
```

All 6 tokens must be emitted in the `<style>:root {}` block alongside `--paper`, `--ink`, `--crema`. After this change, the cold grey defaults in `app.css :root` are never visible on any shop with branding set.
